Este estándar sugiere que una buena ERS debe ser:
- *Correcta*: Todos los requisitos deben reflejar necesidades reales
- *No ambigua*: Cada requisito tiene una única interpretación.
- *Completa*: Incluye todos los requisitos necesarios.
- *Verificable*: Es posible comprobar cada requisito.
- *Consistente*: No hay conflictos entre los requisitos.
- *Modificable*: Se puede actualizar fácilmente.
- *Clasificada*: Organizada de manera lógica
